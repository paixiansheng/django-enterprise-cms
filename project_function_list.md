# Django企业级用户中心+文章CMS 功能清单（必做+可选）
## 一、 核心必做模块（优先级最高，必须完整实现）
### 1.  用户模块（users）
-  自定义用户模型：继承AbstractUser，扩展手机号、头像、生日等字段
-  多方式登录：手机号+密码、邮箱+密码、验证码快捷登录
-  用户信息管理：头像上传（压缩/尺寸限制）、个人资料CRUD、密码重置（邮箱/手机验证）
-  用户状态管理：管理员禁用/启用普通用户
-  登录日志：记录登录IP、时间、设备、登录状态，支持后台查询

### 2.  RBAC权限模块（rbac）
-  核心模型：用户（User）、角色（Role）、权限（Permission）
-  关联关系：用户-角色（多对多）、角色-权限（多对多）
-  权限校验：
  -  页面级：未授权用户拦截后台指定路由（中间件实现）
  -  按钮级：模板中根据权限显示/隐藏操作按钮（自定义模板标签）
  -  接口级：DRF接口权限校验（自定义权限类）
-  权限分配：角色CRUD、权限CRUD、用户-角色分配、角色-权限分配

### 3.  文章内容模块（articles）
-  基础模型：分类（Category）、标签（Tag）、文章（Article）
-  文章CRUD：标题、正文、摘要、分类、标签、发布状态（草稿/已发布）、封面图
-  增强功能：
  -  富文本编辑：集成TinyMCE/CKEditor
  -  封面图处理：压缩、缩略图、按日期存储
  -  分页查询：后台/前台文章列表分页
  -  筛选搜索：按分类/标签/时间筛选，标题/摘要模糊搜索
  -  草稿箱：未发布文章自动保存，支持后续编辑发布

## 二、 附加亮点模块（优先级次之，提升项目竞争力，按需实现）
### 1.  异常处理与日志
-  自定义404/500页面
-  全局异常捕获中间件
-  日志记录：错误信息、访问日志存入文件

### 2.  数据缓存
-  热门文章缓存、首页数据缓存
-  缓存过期配置、文章更新时自动清理缓存

### 3.  全文检索
-  集成Django Haystack + Whoosh
-  文章标题/正文检索、关键词高亮、检索结果分页

### 4.  数据统计与仪表盘
-  统计指标：文章发布量、用户注册量、热门文章访问量
-  可视化：Chart.js实现柱状图/折线图
-  后台仪表盘（Dashboard）展示统计数据

## 三、 功能边界说明（避免后续扩充需求导致返工）
1.  不实现复杂前端框架（如Vue/React），仅使用Bootstrap 5快速搭建界面
2.  不对接真实短信服务，使用「本地验证码模拟」实现快捷登录
3.  全文检索使用本地Whoosh，不依赖Elasticsearch（降低部署难度）
4.  数据库默认使用SQLite（本地演示），预留MySQL配置（生产环境）
